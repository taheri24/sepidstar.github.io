# مقدمه

این برنامه به عنوان درگاه تحت وب برای دستگاه های تردد می باشد

# واژگان

Term توضیحات
Application برنامه کامل شامل سمت سرور و سمت کلاینت - معمولا پایگاه داده محور
Frontend توسعه سمت کلاینت با تکنولوژی های لبه انجام شدهReact-TypeScript - توسعه فرانت به دوقسمت هسته و دامنه تقسیم کرده ایم
Backend در حال حاضر توسعه سمت سرور بوسیله C#(OWIN),Nodejs(TypeScript+Expressjs) انجام میشود و از پروتکل REST پیروی می شود
Core هسته سیستم که بصورت مشترک توسط چند دامین استفاده شده است ، امکانات زیادی برای ایجاد یک برنامه پایگاه داده محور می دهد
Domain دامنه سیستم بیشتر روی منطق کسب و کار برنامه تمرکز دارد و توسعه بیشتر مبتنی بر Core می باشد
templatedView قالب از پیش طراحی شده که کار توسعه دامنه تسهیل می بخشد - بیشترین نقش را در روند توسعه دامنه دارد
React نحوه چیدمان عناصر فرانت اند بوسیله این کتابخانه انجام می شود - بعد از templatedView بیشترین نقش دارد
TypeScript به عنوان زبان توسعه فرانت مطرح می باشد
FP برای استفاده موثر از زبان جاوااسکریپت رویکرد ما پیاده سازی توابع خالص و ریاضی محور است - Functional Programming -
AntiIF کدهای دامین نباید میزان زیادی دستورات شرطی داشته باشد - چراکه باعث پیچیدگی آن می شود

# انتخاب React

پروژه ری اکت در زمان بسیار کمی توانسته به عنوان محبوب ترین فریم ورک ساخت سیستم تحت وب مطرح شود در هنگام نگارش این سند ری اکت پس از چهار انتشار به نسخه 16 رسیده است . هرچند جوان است ولی بسیار بالغ بنظر می رسد البته با توجه مینیمال بودن طراحی آن بسیار از قابلیت های خاص برای تولید برنامه های سازمانی ندارد یعنی در هسته وجود ندارد که کاملا درست و اصولی است و این قابلیت ها در یک پروژه جدا بنام Oragnic-UI خلق کردیم . این فریم ورک React را از حالت کتابخانه ای(تابع و کلاس ) به سمت برنامه های سازمانی هدایت کرده است

# انتخاب TypeScript

زبان بسیار مناسب که امکان نوشتن کدهای کاملا منظم و منسجم به برنامه نویسان تحت وب میدهد و نسبت به جاوااسکریپت بسیار خوش ساخت تر و منظم تر است

  • شبیه به زبان های ساختار یافته است
  • امکانات بسیار زیادی برای Clean-Code می دهد
  • برنامه نویسی AOP پشتیبانی میدهد
  • از همه مهمتر Reactرا بسیار بهتر از جاوااسکریپت پشتیبانی میکند - توکار
  • VS-Code آنرا به صورت بسیار کامل پشتیبانی میکند

# فریم ورک OragnicUI - Core

این فریم ورک در قالب یک ماژول درآمده است ، هدف آن ایجاد سریعتر صفحه های برنامه های سازمانی از روی الگوهای پیش ساخته templatedView می باشد

  • استفاده موثر از الگو از پیش طراحی شدهtemplatedView - هدف اصلی ما هدایت توسعه دهنده دامنه به استفاده از الگو ها می باشد
  • پیاده سازی صفحه هات single-view , list-view - با رویکرد آسان و مهندسی شده
  • انسجام بیشتر کدها
  • حداقل رساندن زمان ساخت باندل build time for bundle
  • یادگیری آسان - رسیدن به نسخه Production بدون کار اضافه
  • انعطاف پذیر بودن و عدم محدودیت برای React

جهت اطلاعات بیشتر به سیستم مستندات با فرمتPowerPoint مراجعه کنید
